PHASE 1 — LOCAL VALIDATION (Do this before anything else)
===========================================================

Step 1: Install dependencies
------------------------------
  cd disasterman-v2/
  pip install torch --index-url https://download.pytorch.org/whl/cpu
  pip install -r requirements.txt

Step 2: Train ZoneScorerNet (one-time)
----------------------------------------
  python agents/train_zone_scorer.py
  # Expected output: "ZoneScorerNet saved to agents/zone_scorer_weights.pt"
  # Takes ~8 seconds on CPU

Step 3: Start the server locally
----------------------------------
  python main.py
  # Server starts at http://localhost:7860

Step 4: Verify endpoints
--------------------------
  # Health check
  curl http://localhost:7860/health
  # Expected: {"status":"ok","env":"DRC-Env v1.0","sessions_active":0}

  # Task list (judges verify 3+ tasks)
  curl http://localhost:7860/tasks
  # Expected: JSON with task_1, task_2, task_3

  # Test episode
  SESSION=$(curl -s -X POST http://localhost:7860/reset \
    -H "Content-Type: application/json" \
    -d '{"task_id":"task_1"}' | python3 -c "import sys,json; print(json.load(sys.stdin)['session_id'])")
  echo "Session: $SESSION"

  curl -X POST http://localhost:7860/step \
    -H "Content-Type: application/json" \
    -d "{\"session_id\":\"$SESSION\",\"action\":{\"action\":\"deploy_team\",\"to_zone\":\"A\",\"units\":3}}"

Step 5: Run baseline agent
----------------------------
  export GROQ_API_KEY=your_key_here   # get free key at console.groq.com
  python inference_v2.py
  # Expected: scores ~0.75-0.90 / ~0.45-0.65 / ~0.25-0.45 for tasks 1/2/3
  # Takes 2-4 minutes (LLM calls)

Submission description template:
---------------------------------
DRC-Env is a multi-zone disaster relief coordination environment where
an AI agent deploys rescue teams, routes supplies, and makes airlift
decisions under time pressure, resource scarcity, and false SOS signals.

Our baseline agent uses a 4-stage pipeline:
1. PyTorch ZoneScorerNet — small MLP (6->16->1) trained on 50K synthetic
   examples, scores zones in <1ms using domain-knowledge features.
2. Triage Agent — LLM detects false SOS signals and deadline alerts.
3. Planner Agent — LLM generates 3-step lookahead resource allocation.
4. Action Agent — LLM selects final action with hard constraint validator.

Anti-hallucination: explicit constraint injection + post-LLM validator
+ deterministic fallback. All LLM stages use llama-3.3-70b-versatile.

3 tasks: easy (1 zone), medium (5 zones + road blocks), hard (10 zones
+ false SOS + dam break event + cascading failures).

================================================================
TROUBLESHOOTING
================================================================

Problem: /baseline returns 503
Fix: GROQ_API_KEY not set in HF Secrets.
     Space -> Settings -> Repository secrets -> Add GROQ_API_KEY

Problem: HF Space shows "Error" status
Fix: Click Logs tab. Most common: missing file in repo.
     Re-check that all files from disasterman-v2/ were copied.

Problem: Docker build fails on torch install
Fix: The Dockerfile installs CPU torch from a special index URL.
     Check that this line is present in Dockerfile:
     RUN pip install --no-cache-dir torch --index-url https://download.pytorch.org/whl/cpu

Problem: inference_v2.py hangs or times out
Fix: Groq rate limits hit. STEP_DELAY=0.5 in inference_v2.py gives buffer.
     For parallel runs (run_all_parallel), consider running tasks sequentially
     if rate limits are hit: call run_task("task_1"), run_task("task_2"), etc.

Problem: HF Space is sleeping / returns 503 after idle period
Fix: Free-tier Spaces sleep after inactivity. Wake it with:
     curl https://krishpotanwar-disaster-relief-env.hf.space/health
     Wait ~30 seconds and retry.

Problem: Submission portal not open yet
Fix: Portal opens March 28, 2026. The Space is already deployed — just
     wait for the portal to open, complete Phase 4 checklist, then submit.
